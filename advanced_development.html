TOELICHTING

Ik heb deze specialisatie gekozen zodat ik mijn portfolio wat levendiger kan maken door animaties. Om deze reden heb ik me vooral gefocust op visuele libraries, zoals AnimeJS en GSAP.Ik heb besloten om GSAP (Green Sock Animation Platform) te gebruiken voor mijn laatste opdracht voor Advanced Development, omdat het compatibel is met zowat alle browsers, erg snel is en vele opties heeft. Ik had graag meer willen uitwerken, waaronder DrawSVG waarmee ik mijn headers getekend wilde hebben, maar helaas zijn dat betaalde functies van GSAP. Ik heb deze library uitgewerkt in mijn portfolio, zodat de specialisatie ook daadwerkelijk iets toevoegt aan mijn portfolio. Dit heb ik gerealiseerd met standaard animaties, scrub animaties en scroll animaties.

GSAP


GSAP is een snelle, robuuste library waarmee je zonder al teveel code gemakkelijk voorwerpen kunt animeren. GSAP is gemakkelijk onder te knie te krijgen, ik zal even uitleggen hoe het werkt.


                gsap.from('#nslogo', {
                    opacity: 0,
                    duration: 1,
                    rotation: 360,
                    scale: 0.1,
                    delay: 2.5,
                    ease:"expo.out"
                  });

Dit is hoe een standaard GSAP animatie eruit ziet. In dit geval, is het de animatie van mijn logo in de header op mijn Portfolio. Ik zal even uitleggen welk element wat doet.


                gsap.richting('geanimeerde object', {
                    opacity: doorzichtigheid van het object
                    duration: lengte van de animatie
                    rotation: rotatie van het object
                    scale: de grootte van het object
                    delay: de vertraging tot de animatie start
                    ease: hoe de animatie verloopt (verschillende effecten)
                  });

Zoals je kan zien, heeft GSAP dus behoorlijk veel mogelijkheden. Van de richting van de animatie (eindpositie van animatie of juist de beginpositie) tot de manier hoe de animatie verloopt, de mogelijkheden zijn eindeloos. Dit is kort maar krachtig hoe GSAP in dit geval werkt. Er zijn nog veel meer mogelijkheden, maar als ik die allemaal uitwerk, dan zijn we hier nog wel even.


Om meer voorbeelden van GSAP te zien, kan je op mijn Portfolio kijken. Hierin heb ik verschillende soorten animaties gemaakt, zoals scrollanimaties, scrubanimaties en meer! Klik daarvoor op de knop op de header of de knop hieronder.

AnimeJS

Ik heb AnimeJS gekozen voor mijn top drie, omdat ik mijn portfolio levendiger wil maken. Met AnimeJS kan je vloeiende animaties maken van tekst, vormen en meer, precies waar ik naar op zoek was voor mijn Portfolio. Echter vond ik GSAP wel prettiger om mee te werken, waardoor hij niet de gekozen library was voor mijn eindopdracht.

Het is me helaas niet gelukt om Slick werkend te krijgen. Er zou nou een caroussel te zien moeten zijn, maar zelfs met hulp van andere medestudenten, krijg ik het niet werkend.


BideoJS

Bideo.js

Met BideoJS kan je video's als background video gebruiken (vandaar de naam Bideo, als in Background Video). Er waren weinig tutorials over te vinden, maar uiteindelijk heb ik het toch werkend gekregen. Kennelijk werkt deze niet zo goed samen met mijn stylesheet, maar toch leuk het mij een leuk idee om uit te proberen.

Reflectie

Ik vond deze specialisatie erg nuttig, omdat libraries coderen een stuk gemakkelijker maakt. Ik had verwacht dat het wat moeilijker zou zijn, aangezien ik nog geen ervaring had met JavaScript, maar achteraf gezien hadden ze het niet gemakkelijker kunnen maken. Ik heb geleerd hoe veel verschillende libraries werken en ik had met mijn basiskennis van JavaScript deze POCs niet zonder de libraries na kunnen maken. Ik ga libraries zeker vaker in de toekomst gebruiken en ik ben blij dat ik alvast wat ervaring op heb kunnen doen.
